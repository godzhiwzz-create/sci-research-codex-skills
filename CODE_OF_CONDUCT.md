# Code of Conduct

## Our commitment

我们希望这个仓库成为一个严谨、尊重事实并保护研究参与者的协作空间。参与 Issue、Pull Request、评审、讨论或其他仓库活动，即表示同意遵守本准则。

## Expected behavior

- 尊重不同经验、学科背景、语言和研究路线。
- 批评方法、证据和实现，不攻击个人或揣测动机。
- 清楚区分事实、推断、假设、模拟结果和未验证结论。
- 保护凭据、个人信息、未公开数据、匿名评审材料和受限研究资产。
- 发现错误时及时更正，并给出可复现的证据或最小示例。
- 尊重署名、许可证、引用和贡献归属。

## Unacceptable behavior

- 骚扰、歧视、威胁、侮辱或持续的人身攻击。
- 公开他人的私密信息、凭据、未发布结果或匿名身份。
- 伪造结果、隐瞒关键限制，或把会话记忆和模拟数据包装为项目证据。
- 绕过安全边界、诱导未授权执行，或利用维护流程破坏他人文件和工作区。
- 在维护者明确要求停止后继续扰乱讨论或重复施压。

## Scope

本准则适用于仓库、Issue、Pull Request、代码评审、Release、Pages，以及代表本项目参与的其他公开或私密协作渠道。

## Reporting and enforcement

普通分歧应先围绕证据在原讨论中解决。涉及骚扰、威胁、隐私、凭据或其他敏感细节时，请使用 [GitHub private vulnerability reporting](https://github.com/godzhiwzz-create/sci-research-codex-skills/security/advisories/new)，并在标题前加 `[Conduct]`。不要在公开 Issue 中披露事件细节。

维护者会尽量保密核查报告，并可根据严重程度编辑或删除内容、暂停互动、关闭讨论、拒绝贡献或限制仓库访问。故意虚假或报复性报告同样违反本准则。

本准则参考了 [Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) 的社区协作原则，并针对科研证据与数据保护场景进行了简化。
