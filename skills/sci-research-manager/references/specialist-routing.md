# Optional Specialist Routing

The bundled `sci-*` skills are the portable default. If the environment already provides a narrower specialist skill, select only the smallest task-matching set, read its instructions, and keep `sci-research-manager` responsible for state/evidence boundaries.

| Need | Bundled default | Optional installed specialist examples |
|---|---|---|
| Literature discovery/indexing | `sci-literature-manager` | `nature-academic-search`, `literature-review` |
| Full paper understanding | `sci-paper-reader` | `nature-reader`, `cv-paper-reading` |
| Experiment protocol/statistics | `sci-experiment-manager` | `experiment-results-planning`, `statistical-analysis`, data-validation skills |
| Claim/submission management | `sci-paper-manager` | venue-specific writing/data/citation skills |
| Evidence audit | `sci-result-auditor` | data-quality/analysis-validation skills |
| Manuscript prose | `academic-manuscript-writing` | `cv-paper-writing`, `nature-writing`, polishing/rebuttal skills |
| Archive/delete review | `sci-asset-manager` | none required |
| Figures/slides/docs | external specialist | research-figure, presentation, PDF, DOCX, or spreadsheet skills |

Do not require optional skills for core operation. Do not let a specialist output bypass experiment provenance, claim calibration, user authorization, or project-local rules.
