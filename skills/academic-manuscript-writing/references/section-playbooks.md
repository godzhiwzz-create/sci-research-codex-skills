# Section Playbooks

## Abstract
State the problem, the key gap, the main evidence chain, the strongest finding, and the boundary of the claim.
Avoid a long literature review or a detailed result dump.
Typical order: problem -> question -> evidence -> takeaway -> scope.

## Introduction
Define the question and motivate why it matters.
Use the introduction to set up the evidence chain, not to narrate all results.
Typical order: context -> gap -> research question -> approach -> contribution.

## Setup / Problem Framing
Define the experimental protocol, metrics, and terminology needed to read the results.
Keep it minimal and operational.
Avoid explaining results that belong in the observation or mechanism section.

## Observation / Results
Present the phenomenon compactly.
Show the pattern that needs explaining, then stop.
Avoid exhaustive cell-by-cell narration unless the user explicitly wants it.

## Mechanism / Analysis
Compare candidate explanations against the observed pattern.
Use the strongest evidence here and explain why weaker mechanisms fail.
Make the section argumentative, not descriptive.

## Discussion
Explain what the evidence chain changes for interpretation and design.
State limitations, scope, and implications.
Avoid re-listing all numbers already shown earlier.

## Conclusion
Restate the narrow claim supported by the paper.
Do not introduce new evidence or a new experiment narrative.

## Supplementary Material
Use for stable but non-essential details, repetitive numbers, or supporting checks that would interrupt the main argument chain.
